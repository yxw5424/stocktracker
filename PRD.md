## 1.2 分组小结

**A 在线量化**：回测与 A 股数据最强，但**多数要写代码**；唯一零代码的果仁仍是**数值硬条件**；定位是"研发/自动执行"而非"盘中语义盯盘"；果仁"策略订阅发买卖指令"是**变相荐股反面教材**。

**B 券商/行情终端**：用户基数与数据壁垒最高，实时监控+多渠道告警工程成熟，且**问财、华泰已把 NL 做进盯盘并在向 Agent 演进**（修正 v0.1）。短板是券商系强绑自家账户、中立性弱；问财曾因"AI 荐股未打标"被点名——**大厂也踩线**。

**C 国际平台**：工程标杆（TradingView 告警生态、LEAN 开源回测、Composer 无代码+NL），验证"NL→可执行/可告警"散户付费成立；但**几乎都不覆盖 A 股**（无资金流/龙虎榜/题材/涨跌停/本土新闻），且 AI 多用于"帮写代码/告警后解释"而非语义触发。Quantopian 教训：**实时/实盘运维与合规成本最高，且变现别寄托于策略分成**。

**D AI/LLM 投研**：与本赛道最近。持牌系（华泰/妙想）有**合规与信任背书**（妙想已过网信办备案、可溯源）；开源 TradingAgents 提供**多 agent 研判蓝本**。但：问财/妙想/雪球更偏"选股/问答/研报"，研判型 agent **非实时、不可复现、token 成本高**；能力上限现实——FinGPT 情绪分类强（F1 87%+）但**方向预测仅 45–53%**（略高于抛硬币）。

## 1.3 空白点分析（修订版：更克制）

| 能力维度 | 谁做得最好 | 真实缺口（修订后） |
|---|---|---|
| ①盘中**实时**盯盘 | 富途(14类异动)、华泰(365天)、TradingView(服务器端告警)、**问财(自选监控)** | 主流是**预置异动模型/硬阈值**;"LLM 逐时段读盘面+资金+新闻综合研判"尚不普及,**但这块越成立越踩投顾红线**(§3) |
| ②**定时**主动研判/复盘 | TradingAgents(辩论)、妙想/雪球(深度研究) | 开源框架非实时/不可复现;持牌系偏"问答/研报"。**是相对空白,但对有自研大模型的大厂是数周工程量,非壁垒** |
| ③**NL 提示词规则** | 华泰、问财、StockTalker | 都偏 **"NL→硬阈值/可执行条件/横截面选股"**;"利好兑现却高开低走→见光死"这类**情景+情绪复合语义常驻触发**尚无成熟产品——**但这恰恰是 LLM 最不擅长、且最像投顾的部分**(§4.7、§3) |

> **修订后的诚实结论**：曾经写的"组合空白格无人覆盖"**站不住**——心智已被占位、功能可被数周复制。真正还**没人做透**的只有"情景+情绪复合语义的盘中常驻触发"，而它同时是 **(a) LLM 可靠性最差、(b) 合规风险最高、(c) A 股免费数据最难支撑实时**的三重难点。所以它不能当 MVP 的"核心卖点"，而应作为**护城河方向上的实验**（§2.3、§4.7）。

# 二、产品 PRD

## 2.0 一句话定位（修订）

**面向中国散户的"自然语言客观事件盯盘器"：你用大白话写盯盘意图（"放量突破平台""炸板""加速赶顶"），我们用 LLM 在配置期把它翻成可计算的硬指标 DSL（低风险、高价值），盘中用确定性引擎常驻盯你的跨券商自选股，命中即提醒；只报客观事实、不给方向建议、不替下单。** 情景语义研判（如"见光死"）作为**实验功能**，仅在合规裁决允许、且离线评测达标后灰度开放。

差异化一句话（修订）：不再宣称"NL 盯盘无人覆盖"（已被占位），而是押注**对手抄不走的沉淀型资产**——见 §2.3。

## 2.1 目标用户

| 分层 | 特征 | 核心诉求 |
|---|---|---|
| 核心 | A 股新手/进阶，爱题材概念股，盘中没法盯盘，**有满仓梭哈冲动** | 替我盯客观事件、命中就提醒、帮我"等一等再决定" |
| 次级 | 技术派/打板者 | 把模糊盘感落成**可复用的常驻规则** |
| 排除(MVP不服务) | 量化开发者、机构、期货/期权/跨境为主 | 已有聚宽/掘金/真格/QuantConnect |

> 对核心画像的合规约束：用户"有满仓梭哈冲动"意味着产品**不得强化追涨杀跌**，须内置冷静机制与能力局限告知（§3.3）。

## 2.2 痛点

1. **盯不过来**：自选多、要上班，错过变盘点。
2. **写不出条件**：脑子里是"见光死/放量突破"这种模糊盘感，传统工具只让填"涨幅>3%"。
3. **传统告警太吵**：纯阈值误报多→关推送→工具失效（PagerDuty：误报是生死线）。
4. **AI 形态错位**：问财/妙想偏"问一次答一次"；常驻盯单票+情景命中主动叫我，仍不普及。
5. **缺主动研判/复盘**：盘中/盘后缺一个按节奏主动提示"该关注啥"的助手。

## 2.3 差异化与护城河（重写：从"功能点"到"沉淀资产"）

审稿意见核心：**功能点（NL 盯盘、每小时研判）会被大厂数周复制，且"中立/只提示"是合规被迫、对用户是劣势。** 因此差异化必须收敛到**随使用时间增厚、对手无法一次性抄走**的资产。四个候选护城河，按可守度排序：

### 护城河①：跨券商中立的自选聚合 + 用户私有规则库（最可守）
- **是什么**：用户在一处管理**不分券商**的自选股与**个人 NL 规则库**；规则与历史告警是用户资产，可导出、跨设备。
- **为何对手抄不走**：华泰/同花顺的盯盘**强绑自家账户/生态**，结构上不会替竞争对手的持仓做中立聚合；这是**商业模式冲突**而非技术差距。用户规则库越用越厚，迁移成本越高。
- **诚实校准（采纳审稿意见）**：中立的代价是**下单断点**——提醒后用户要切回券商 App。所以中立**只对"跨多家券商/多账户、且非争分夺秒"的用户是优势**；对纯打板用户是劣势。定位话术不吹"中立=快"，而是"中立=你的规则与自选不被某一家锁死"。

### 护城河②：用户标注沉淀的评测集 + 个性化校准（数据飞轮，提前到 MVP）
- **是什么**：每条告警都可被用户标"有用/没用/误报"，沉淀为**带 A 股盘面上下文的私有评测集**；用于回归测试、并对**该用户**的灵敏度做个性化校准。
- **为何可守**：这是**冷启动后才长出来的数据**，对手即便复制功能也复制不了你的标注资产。**修订：把"评测集"从 v0.1 的 P1 提到 MVP（P0）就开始采集**——否则启动期完全暴露在大厂功能覆盖下，没有任何对手抄不走的东西。

### 护城河③：把模糊盘感翻成规则的"教学体验"与高质量模板口碑
- **是什么**：新手不会写规则（审稿意见正确指出这是核心交互门槛），我们做**引导式规则生成**（选盘感→反问澄清→回译→回放）+ 一套经实测、低误报的**官方情景模板**。
- **为何可守**：模板的**口碑与命中质量**是运营+数据沉淀的结果，不是一次性功能。

### 护城河④（弱、且高风险）：情景+情绪复合语义研判
- 这是 v0.1 当作"核心卖点"的点。**修订后明确：它是研究方向上的实验，不是 MVP 卖点**，原因：(a) LLM 方向判断仅 45–53%（§1.2）；(b) 它最像投顾、合规风险最高（§3）；(c) A 股免费数据撑不起秒级实时（§4.6）。差异化越成立、合规风险越高——这条结构性张力**只能靠产品形态化解**（去方向化或持牌合作），不能靠免责声明。

> 护城河取舍结论：**MVP 押①②③（中立聚合+私有规则与标注资产+教学/模板口碑），把④作为受合规闸门约束的实验。** 并需在用户研究中验证①②③**是否值得用户付费**（审稿意见的开放问题，列入 §7 验收）。

## 2.4 提示词规则引擎设计（核心章节，已按审稿意见收口）

> 总原则：**把"聪明"放配置期，把"确定"放运行期。** LLM 在配置期帮你拆解、反问、回译；运行期 MVP **优先纯硬指标**，语义确认是受闸门约束的实验。

### 2.4.1 用户怎么写规则（配置期四步，借鉴 Zapier）
1. **解析**：LLM（函数调用/JSON Schema 约束）把 NL 拆为结构化规则对象（硬指标 DSL + 可选语义判断 + 动作 + 冷却）。
2. **回译（强制）**：用大白话+可视化展示"系统理解成了什么"。**审稿意见警示：回译≠正确，用户点确认≠真懂**——故回译须做两件事降低"被包装得很可信的错误"：
   - **显式列出每个被量化的阈值与默认值**（如"'最近'=5 个交易日，可改"），让分歧点可见；
   - **强制走回放**（下条）作为客观校验，不允许"看完回译直接启用"。
3. **澄清/弃权**：解析不确定**主动反问**而非乱猜；无法量化的语义**默认不进 MVP**，若开启则醒目标注"由 LLM 判断（实验功能，可能误差大）"。
4. **回放 + 影子模式**：用近 N 个交易日数据**回放**展示历史触发点（语义段的回放须做 point-in-time 对齐，见 §4.7 防前视偏差）；再以"只记录不推送"影子运行 1–2 日，用户看到"如果开了会推几条"才正式启用。

**护栏**：提供模板降冷启动；单条规则**小而清晰**（否定/时序/组合极易错），引导拆成多条；**用户提示词=不可信输入**，做注入隔离（§3.2）。

### 2.4.2 两段式架构（成本与准确性核心）

```
配置期(一次性, LLM重):
  NL规则 ──LLM解析(填槽,不下结论)──► 规则对象{硬指标DSL + (可选)语义判断 + 动作 + 冷却}
                                       └─► 回译(列出阈值/默认值) ─► 历史回放(PIT对齐) ─► 影子模式

运行期(盘中):
  第一段 硬指标初筛(确定性, 无LLM, 毫秒级)  ← MVP 仅此段
     逐分钟bar算量价/形态布尔表达式 ──[绝大多数否决, 成本≈0]
        │ 命中候选
        ▼
  第二段 LLM语义确认(实验功能, 受合规闸门, 低频可缓存)
     结构化输出{触发|不触发|不确定} + 理由 (注意: confidence不作真实准确率闸门, 见§4.7)
        │ 触发 且 通过去重/合并/冷却
        ▼
     告警(强制免责标识 + 仅客观盘面/事件描述)
```

**为何这样分**：硬指标用近零成本挡掉绝大多数时刻（Datadog）；LLM 只在稀疏候选跑、且只"填槽+二选一+给理由"，降低"沉默错误"（Text-to-SQL）。**修订：MVP 默认只跑第一段**，第二段在合规允许 + 离线评测达标后才灰度。

### 2.4.3 规则对象 Schema

```json
{
  "rule_id": "r_001",
  "raw_nl": "放量突破平台、利好兑现但高开低走，提醒我可能见光死",
  "symbol_scope": ["sh600000"],
  "enabled": true,
  "hard_filter": {                       // 第一段：便宜、确定、可量化、可静态校验
    "logic": "AND",
    "conditions": [
      {"indicator": "volume_ratio",  "op": ">=", "value": 2.0, "window": "vs_ma20"},
      {"indicator": "price_breakout","op": ">",  "ref": "box_high_20d"},
      {"indicator": "intraday_shape","op": "==", "value": "high_open_low_close"}
    ]
  },
  "semantic_judge": {                    // 第二段：实验功能；受合规闸门 §3
    "enabled_experimental": false,       // MVP 默认关闭
    "question": "最近利好是否已被price-in?是否呈见光死特征?",
    "evidence_sources": ["news_3d_pit", "intraday_curve", "capital_flow"],  // _pit = point-in-time
    "output_schema": {"verdict":"enum[trigger,no_trigger,uncertain]","reason":"string"},
    "trigger_when": "verdict == trigger",   // 注意：不以LLM自报confidence作闸门(§4.7)
    "min_offline_precision_to_enable": 0.70 // 离线评测精确率达标前不得对外开启
  },
  "action": {
    "channel": ["app_push","wechat"],
    "message_template": "【规则命中·非投资建议】{symbol} 触发『{rule_name}』：客观盘面：{facts}",
    "must_include_disclaimer": true,
    "forbid_directional_words": true     // 拦截"建议买/卖/加仓/目标价/抄底"
  },
  "noise_control": {                     // PagerDuty 式一等公民，必填
    "cooldown_sec": 3600,
    "dedup_key": "symbol+rule+signal_bucket",
    "max_alerts_per_day": 3,
    "merge_window_sec": 300
  },
  "audit": {"created_by":"user","model_version":"...","translated_back":"...","pit_aligned": true}
}
```

**关键设计**：`hard_filter` 用**封闭词表 DSL**，LLM 只能填槽选指标/阈值、不能生成自由代码（安全+可静态校验）；`semantic_judge` 默认关闭、需达**离线精确率门槛**才可开（§4.7）；`noise_control` 必填且默认值合理。

**预定义硬指标 DSL（封闭词表，节选）**：`volume_ratio`、`price_breakout`（箱体/均线/前高）、`pct_change`、`intraday_shape`（high_open_low_close/surge/dump）、`slope`（分时斜率，支持"斜率激增"）、`limit_status`（涨跌停/炸板*）、`turnover`、`gap`、`ma_cross`、`amount`。
> *炸板的时效性受数据延迟约束（审稿意见正确指出分钟 K 抓不到封板-炸板瞬时过程）——见 §4.6，MVP 对此类**秒级信号明确标注"尽力而为、可能滞后"**，不承诺实时。资金面（DDX/DDY）、龙虎榜为付费/受限数据，P1 合规接入。

### 2.4.4 去重/冷却/回放/灰度（PagerDuty + Datadog）

| 能力 | 做法 | 借鉴 |
|---|---|---|
| 回放(Backtest-lite) | 近 N 日重放;**语义段必须 point-in-time 对齐历史新闻/情绪,严禁用当前快照(防前视偏差)** | Datadog 离线评估 |
| 去重 Dedup | 按 `symbol+rule+signal_bucket` 折叠 | PagerDuty |
| 合并 Merge | `merge_window` 内同类合并 | PagerDuty |
| 冷却 Cooldown | 同信号静默期 + 每日上限封顶 | PagerDuty |
| 升级 Escalate(P2) | 同票多规则命中→合并"重点关注" | PagerDuty |
| 影子/灰度 | 新规则"只记录不推送"1–2 日,达标再放量 | Datadog 先离线后上线 |

## 2.5 技术架构

### 2.5.1 分层

```
Frontend (GitHub Pages / 静态SPA)
  自选管理 · 分时/日K切换 · 均价线 · 规则配置/回译/回放 · 告警流 · 研判卡片
       │ HTTPS / WebSocket
后端服务 (Python)
  数据层(行情/资金/新闻采集+缓存+防封+多源冗余)
     → 规则引擎(硬指标DSL执行 + 去重/冷却/合并)
     → LLM编排(配置期解析 / 运行期语义确认[实验] / 定时研判[P1])
     → 调度&自适应节奏  →  告警分发(多渠道)
  存储：规则库 / 评测集 / 告警日志 / 行情缓存
```
> **托管约束**：纯静态 Pages **无法跑常驻进程**，常驻监控/LLM 编排须由后端（个人服务器/轻量云函数+定时任务）承载，Pages 仅做前端。这是从 MVP 走向常驻 agent 的关键架构变化。

### 2.5.2 数据层（直面单点风险，审稿意见重点）
- **现状**：Python + 新浪源（东财对本机不可达）抓分时/日 K；已做防封（合并请求、当天日 K 缓存、随机抖动、指数退避识别限频、排周末）。**README 已自承：非逐笔实时、秒~分钟级延迟、会限频、源站改版即失效。**
- **修订后的诚实定性**：**整条常驻盯盘押在非官方接口上 = 数据中断即核心承诺违约**。这既是工程命门，也是合规命门（§3.4 数据商用合规）。
- **必做（从"灾难"变"有预案的降级"）**：
  - **多源冗余**：≥2 个独立数据通道，主源失效自动切备源；
  - **降级 SLA**：源不可用时，产品状态从"实时盯盘"显式降级为"尽力而为/延迟监控"，**在 UI 与推送中明确告知用户当前数据时效**，而非静默给出可能过期的提醒；
  - **付费/合规源迁移路径**：实测券商行情 API / 持牌数据商接入与**成本测算**（这很可能颠覆 $5/月定价假设，见 §3.4）；
  - **秒级信号标注**：炸板/瞬时拉升类**明确标注可能滞后**，不承诺逐笔实时。

### 2.5.3 LLM 编排
三类调用：①规则解析（配置期，强模型）②语义确认（运行期，小/快模型+缓存+低温，**实验功能**）③定时研判（P1，多 agent 流水线，参考 TradingAgents 但裁剪为低频/可缓存/可复现）。统一 prompt 隔离 + 注入防护 + 结构化输出 + **可溯源审计日志**（模型版本、输入快照、输出理由——对标妙想"可溯源"建立信任）。

### 2.5.4 调度/自适应（含成本护栏，审稿意见重点）
- 交易日历驱动（排非交易时段/周末/节假日）。基础：盯盘分钟级；定时研判每小时；收盘复盘 1 次。
- 自适应加频：某票斜率激增/放量临时提频。**审稿意见警示：自适应加频会在最贵的时刻放大 LLM 调用**（妖股盘中频繁命中硬指标，候选并不稀疏）。故加频**只提高硬指标轮询频率，不等比提高 LLM 调用**；语义确认仍受冷却/每日上限/预算闸门约束（§4.7）。

**P1（强化质量与受闸门的差异化）**
- 语义确认（实验）：**离线精确率达门槛 + 合规裁决允许**后灰度；缓存/投票/评测集回归。

- 资金面/龙虎榜合规接入；付费/官方数据源迁移与成本实测。
- 规则市场/模板 v1（含内容审核）。

**P2**：规则市场 UGC + 升级合并；个性化冷静机制；多账户中立同步增强；模型分级/批处理降本。

## 2.8 指标（含可证伪的上线门槛，审稿意见重点）

**北极星**：**有效告警采纳率 = 用户标"有用/认可"的告警 / 总告警**（直接反映"规则是否真帮到用户"，天然抑制误报刷量）。

| 维度 | 指标 | **上线红线（可证伪，建议初值，需用真实数据校准）** |
|---|---|---|
| 质量(生死线) | 纯硬指标规则误报率 | **≤ 20%**(影子模式实测,不达标不放量) |
| 质量 | 语义确认(实验)旗舰用例**离线精确率** | **≥ 0.70 才允许对外灰度**;否则保持关闭 |
| 质量 | 弃权率(uncertain 占比) | 设上限,过高说明规则不可量化,应引导用户改写 |
| 成本 | 初筛挡掉比例 | **≥ 95%**;低于则该规则不得启用语义段 |
| 成本 | 单用户单日 LLM 调用 / 成本 | **设硬上限**(见 §4.7);超限按预案降级 |
| 价值 | 人均生效规则数;创建→回放→启用转化 | 趋势向好 |
| 留存 | 次周/次月留存;盘中活跃天数 | 趋势向好 |
| 信任 | 免责标识曝光率;反馈中"对"比例 | 趋势向好 |

> 刻意**不把"荐股准确率/收益率"作为指标**——既因能力上限（FinGPT 45–53%），更因合规（不得承诺收益）。**"可接受误报"必须落到上面的数字，不能停留在 v0.1 的"校准后可接受"这种不可证伪表述。**

# 三、可行性、合规风险与对策

## 3.1 Gate-0：合规可行性裁决（立项前必须先做，审稿意见最高优先级）

**为什么是 Gate 而非检查项**：核查《证券、期货投资咨询管理暂行办法》《证券投资顾问业务暂行规定》确认——**通过软件/通讯工具对"特定证券"提供分析意见、预测、买卖/选择/时机建议并直接或间接获取利益，即属证券投资咨询业务，须经证监会许可，任何单位和个人不得擅自经营。** 本产品对特定个股输出"见光死/可能启动/警惕出货"这类**带方向暗示的情景研判**，即便包裹成"客观盘面+规则命中"，监管极可能认定为**变相投顾**（问财已因"AI 荐股未打标"被点名；持牌的九方智投 2025 年仍因误导性宣传领罚——**无牌做更接近荐股的事风险更高**）。这是**能不能存在**的问题，必须最先裁决。

**裁决产出两条路径之一（由证券合规法务出具书面意见）**：

- **路径 A：彻底去方向化（默认，无牌可行）**——MVP 只输出**纯客观事实**（"放量 2 倍突破近 20 日箱体""涨停后炸板"），**绝不给情景标签与解读**（不出现"见光死/可能启动/出货"）。LLM 仅用于**配置期把大白话翻成 DSL**（低风险、高价值）。这一步**同时化解合规、幻觉、成本、数据实时性四个风险**，是审稿意见明确推荐的 P0 重定位。
- **路径 B：持牌合作/导流分润**——若要保留方向性研判，须与**持牌投顾机构合作**（由持牌方出具研判、本产品做工具与分发），或申请相应资质。

> **强约束**：路径 A 未明确放行前，§2.4 的 `semantic_judge` 全部保持 `enabled_experimental=false`；定时研判走"去方向化"版本（只客观描述+风险提示，不给观点）。

## 3.2 其余合规红线与产品化解

| 风险 | 说明 | 产品级化解（非仅免责） |
|---|---|---|
| 措辞替换不脱敏 | "见光死/可能见顶/警惕出货"本身即**时机/方向判断**;执法实质重于形式 | 取决于 Gate-0;路径 A 下**根本不输出这类标签**,而非换词规避 |
| 承诺收益/夸大准确 | 方向预测仅略高于抛硬币(45-53%) | 话术不承诺"判得准";不展示胜率/收益卖点;北极星不用收益率;**强制 AI 能力局限告知** |
| **数据源商用合规** | 新浪/akshare 非官方接口,普遍约束"仅限个人学习研究、不得商用";网安法/反不正当竞争语境下对外收费存被认定不正当获取数据风险 | **核心命门数据正建立在不合规抓取链上**——收费前必须迁移到**官方/持牌付费源**;成本计入定价(§3.4) |
| LLM 误判 | 沉默错误、漂移、偏置、幻觉、**自报置信度校准差** | 结构化输出+理由+弃权;缓存/低温保一致;免责标识;可溯源审计;用户反馈纠错;**confidence 不作真实准确率闸门(§4.7)** |
| 提示注入 | 用户提示词=不可信输入,可诱导越权/荐股 | prompt 隔离;注入检测;诱导荐股一律拒绝并回落合规话术 |
| **规则市场荐股漏洞** | 用户可在规则名/message_template 夹带"抄底信号"等荐股话术,UGC 传播=平台为变相荐股提供渠道 | 规则市场**内容审核**:对规则名与模板做**荐股话术拦截**;禁含买卖信号 |
| **推送渠道合规** | 依赖微信(PushPlus/Server酱)等第三方转发,金融提醒内容易被判违规/封号 | 评估金融内容在各渠道可持续性;多渠道冗余;敏感措辞过滤;必要时自建合规推送 |
| 用户冲动放大 | 目标用户"爱题材股、有满仓梭哈冲动" | 内置**冷静机制**(满仓/追涨/连续告警插入风险提示与"再等等");**强制风险测评/适当性提示**;产品不强化追涨杀跌 |

## 3.3 合规话术、标识与适当性（强制）
- 所有告警/研判**强制前缀**"【规则命中·非投资建议】"或等效标识；**AI 生成内容显著标识**（对标问财已强制"内容由 AI 生成请甄别"）。
- 全局显著免责：本产品为信息与效率工具，不构成投资建议，不保证准确性与收益，决策与风险由用户自负。
- **投资者适当性落地**：面向新手的强制风险测评 + AI 能力与局限告知（2025 监管主题即"AI+防范欺诈"，强调能力局限告知）。
- 关键节点（启用规则、收到告警、查看研判）重复风险提示，保留"客观盘面 vs 用户主观规则"的边界表达。

## 3.4 上线硬门槛与真实成本结构（审稿意见补漏：立项预算的一部分）

| 门槛 | 说明 |
|---|---|
| ICP 备案 / **ICP 许可证(EDI)** | 一旦收费即属**经营性**互联网信息服务,需 ICP 许可证(非个人可办的非经营性备案);无备案接入商不得接入、分发平台不得上架 |
| 软件著作权 / 应用商店**金融类目审核** | 上架需软著与金融类目资质审核 |
| **金融信息服务相关审批** | 股票提醒类可能触发金融信息服务规范 |
| **合规数据源采购** | 官方/持牌行情数据的采购预算——**很可能颠覆 $5/月定价假设**,须重算真实成本结构 |
| AI 生成内容标识 | 监管强制 |

> **定价现实**：当数据从"免费抓取"切到"合规付费"，叠加 LLM 调用、推送、合规与运维，$5/月的对标价**大概率不成立**。立项时必须用**合规成本反推可行定价**，而非先定价再凑成本。

## 3.6 §4.7 LLM 误报、成本、延迟的工程化缓解（逐条对应审稿质疑）

| 审稿质疑 | 工程化缓解 | 残余风险（诚实） |
|---|---|---|
| **LLM 能力上限自相矛盾**：把"见光死/是否 price-in/假突破"交给最易幻觉的判断 | 路径 A 下**这类判断不进 MVP**;若开实验,**离线精确率≥0.70 才灰度**,否则关闭 | 残余:这类博弈判断 LLM 上限本就低,**可能永远达不到 0.70**——则该功能永久关闭,这是可接受结局 |
| **自报 confidence 当闸门是伪精确** | **不用 LLM 自报 confidence 作触发闸门**;改用 (a) 离线评测得到的**该规则类型经验精确率** + (b) 多次投票一致性 + (c) 硬指标强度 作为放行依据 | 残余:投票仍不消除系统性偏差;只能压不能除 |
| **A 股免费数据跑不通实时;炸板抓不到瞬时** | 秒级信号**明确标注"尽力而为、可能滞后"**;多源冗余+降级 SLA;实测付费源迁移 | 残余:免费源下秒级信号本质不可靠,**靠产品话术降级期望**而非假装实时 |
| **成本乐观、妖股候选不稀疏、加频在最贵时放大调用** | (a)自适应**只提高硬指标轮询、不等比提 LLM**;(b)同票同窗多规则**合并一次 LLM 调用**;(c)新闻/情绪**预处理为结构化信号**,运行期只读;(d)**单用户日调用/成本硬上限**,超限降级为纯硬指标并通知;(e)恶意"频繁命中"规则**限流+异常检测** | 残余:极端行情日成本仍偏高;靠上限封顶,代价是高峰期降级 |
| **NL 三重否定+时序复合易解析错;回译被包装得可信** | 回译**列出每个阈值/默认值**;**强制回放**作客观校验;引导**拆成多条小规则**;复杂复合规则给"解析置信度低"警示 | 残余:散户可能懒得校准——靠**默认走影子模式**兜底,启用前先看"会推几条" |
| **语义一致性:bucket 边界抖动;低温不消除幻觉** | bucket 用**带迟滞(hysteresis)的边界**减少临界抖动;同 (symbol,bucket,rule) 结果短期缓存;低温+缓存 | 残余:LLM-as-judge 固有缺陷,**只能压不能除**——故 MVP 优先纯硬指标(确定性,无此问题) |
| **回放语义段前视偏差(用当前快照)** | 历史回放**强制 point-in-time 对齐**(只用触发时点之前可得的新闻/情绪),`audit.pit_aligned=true` 才算有效回放 | 残余:历史新闻 PIT 数据获取有成本;不达标则**禁用语义段回放**,只回放硬指标 |
| **抓数稳定性=产品稳定性单点** | 多源冗余;主备自动切换;**降级 SLA + 用户告知**;付费/官方源迁移路径 | 残余:迁移付费源前,稳定性与合规仍受限——这是 §3.4 的预算决策 |

## 3.7 工程与运营风险
- **实时/实盘运维最难**（Quantopian 先砍实时）：MVP **不做实盘交易**，常驻监控按自选股集合控规模、可降级。
- **成本失控**：硬指标初筛 + 缓存 + 模型分级 + 自适应轮询（非等比 LLM）+ **单用户预算硬上限**兜底。
- **误报疲劳致失效**：降噪一等公民 + 影子/灰度 + 北极星抑制误报 + **可证伪误报红线**（§2.8）。

| 账户绑定 | **强绑华泰** | 绑同花顺生态 | **跨券商中立**(护城河①) |
| 是否下单 | **可直连下单(体验顺)** | 偏选股/问答 | **只提示不下单**(合规被迫,对争分夺秒场景是劣势——诚实承认) |
| 合规背书 | 持牌(强) | 平台+大模型(强) | **无牌→靠去方向化/持牌合作(Gate-0)** |
| 真正可守的差异 | — | — | **中立自选聚合 + 用户私有规则与标注资产 + 新手规则教学/模板口碑**(护城河①②③) |

> **诚实结论**：v0.1 宣称的"组合无人覆盖"**不成立**——心智已被有牌照、有流量、有自研大模型的大厂占住，且"中立/只提示"是合规被迫、对部分场景是体验劣势。**牌照背书是相对持牌系的最大短板，只能靠 Gate-0（去方向化或持牌合作）化解。** 本产品的存活不靠"想到 NL 盯盘"，而靠：①跨券商中立聚合（商业模式上对手不愿做）②用户标注与规则的**数据飞轮**（时间沉淀，抄不走）③新手教学/模板口碑。这些必须在用户研究中验证**值不值得付费**——若不值得，则本产品的商业前提需重估。

## 关键修订一览（供评审快速定位）

1. **Gate-0 合规裁决前置**（§3.1）：去方向化（默认）或持牌合作二选一，决定产品能否存在。
2. **竞品纠错 + 差异化收敛**（§1.0、§2.3、§3.8）：承认 NL 盯盘心智已被占位；护城河收敛到中立聚合 / 用户标注与规则飞轮 / 新手教学口碑，并把**评测集飞轮提前到 P0**。
3. **旗舰"见光死"降级为实验功能**（§2.4、§4.7）：MVP 默认纯硬指标；语义段需**离线精确率≥0.70 + 合规放行**才灰度；**不以 LLM 自报 confidence 作闸门**。
4. **可证伪的上线门槛**（§2.8）：误报率≤20%、初筛挡掉≥95%、单用户日成本硬上限、弃权率上限。
5. **数据单点 → 降级 SLA + 多源冗余 + 付费源迁移成本**（§2.5.2、§3.4）；秒级信号标注"尽力而为"。
6. **上线硬门槛与真实成本**（§3.4）：ICP 许可证/软著/金融类目/AI 标识/合规数据采购——很可能颠覆 $5/月定价。
7. **回放 point-in-time 对齐防前视偏差**；**规则市场内容审核拦截荐股话术**；**推送渠道金融合规评估**；**自适应只提硬指标轮询、不等比提 LLM**。
